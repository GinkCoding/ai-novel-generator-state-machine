# 🧠 深度逻辑审查 - 终极优化方案

## 🎯 **核心洞察**

**问题：** 即使 AI 逻辑审查也只能发现表面问题，更深层的逻辑问题需要专门审查。

**示例：**
- ✅ 表面问题："帝王墓太显眼"（AI 能发现）
- ❌ 深层问题："陈三爷怎么知道胡八一留了话？"（AI 可能忽略）

---

## 🏗️ **6 层深度审查架构**

```
Layer 1: 信息流审查
  └─ 每个角色知道什么？谁告诉谁的？为什么告诉他？

Layer 2: 视角一致性审查
  └─ 每句话是否符合视角？是否有上帝视角混入？

Layer 3: 情感弧线审查
  └─ 情感变化是否有触发？强度是否匹配？

Layer 4: 动机深度审查
  └─ 行为背后的真正动机？是否与角色一致？

Layer 5: 主题一致性审查
  └─ 情节是否服务于主题？是否偏离？

Layer 6: 伏笔追踪审查
  └─ 伏笔是否回收？是否有遗忘的伏笔？
```

---

## 📊 **各层审查详解**

### **Layer 1: 信息流审查**

**审查问题：**
1. 每个角色知道什么信息？
2. 信息是如何传递的？（谁告诉谁的？）
3. 信息传递是否有合理动机？（为什么告诉他？）
4. 是否有信息跳跃？（突然知道了不该知道的）

**典型案例：**
```
原文："陈三爷，你好像知道些什么。"
      "嗯，你父亲十年前来过，留了三句话。"

问题：
- 陈三爷怎么知道胡八一留了话？
- 胡八一告诉陈三爷的？
- 但胡八一为什么要告诉陈三爷？
- 逻辑链断裂！

修复：
补充信息传递渠道：
"你父亲当年告诉我的。他信得过我，因为我是唯一一个既懂守墓人规矩，
又在 701 部门能说得上话的人。"
```

---

### **Layer 2: 视角一致性审查**

**审查问题：**
1. 当前视角角色是谁？
2. 每句话/每个描述是否符合该视角？
3. 是否有视角跳跃？
4. 是否有"上帝视角"混入？

**典型案例：**
```
原文（胡念一视角）：
"陈三爷眼中闪过一丝欣慰。"

问题：
- 这是胡念一的视角
- 但胡念一怎么能看到陈三爷的眼神？
- 视角跳跃！

修复：
改为胡念一能观察到的：
"陈三爷嘴角微微上扬，像是松了口气。"
或改为转述：
"后来我才知道，当时陈三爷眼中闪过一丝欣慰。"
```

---

### **Layer 3: 情感弧线审查**

**审查问题：**
1. 每个角色的情感状态是什么？
2. 情感变化是否有触发事件？
3. 情感强度是否与事件匹配？
4. 是否有情感突变？

**典型案例：**
```
原文：
"陈三爷沉默了很久。'好吧，我告诉你们真相。'"

问题：
- 陈三爷从"隐瞒"到"坦白"
- 情感转变的触发点是什么？
- 没有明确触发事件
- 情感变化突兀！

修复：
添加触发事件：
"陈三爷沉默了很久。他突然从怀里掏出一张泛黄的照片，
递给我。'这是你父亲当年留下的。他信得过我，我也该
信得过你。'"
```

---

### **Layer 4: 动机深度审查**

**审查问题：**
1. 每个角色的行为动机是什么？
2. 动机是否充分支持行为？
3. 是否有更深层的动机？（表面/深层）
4. 动机是否与角色背景一致？

**典型案例：**
```
原文：
"701 部门要灭口所有知情人。"

问题：
- 701 的动机是什么？
- 为什么要灭口？
- 作为国家机构，灭口不符合其使命
- 动机不充分！

修复：
提供深层动机：
"701 不是要灭口，是要保护。被收容的五个人都出现了
异化症状，701 怕他们传染给外界。但方法太极端，
变成了软禁。"
```

---

### **Layer 5: 主题一致性审查**

**审查问题：**
1. 这个故事的核心主题是什么？
2. 每个情节是否服务于主题？
3. 是否有偏离主题的内容？
4. 角色行为是否体现主题？

**典型案例：**
```
主题：探索真相

原文：
团队被动接受陈三爷的信息，没有主动探索。

问题：
- 主题是"探索真相"
- 但主角被动接受信息
- 情节与主题脱节！

修复：
让主角主动探索：
"我拿出父亲留下的笔记，翻到最后一页。'你说的这些，
父亲都写过。但有一件事他不明白，为什么墓里会有蝉鸣？'"
```

---

### **Layer 6: 伏笔追踪审查**

**审查问题：**
1. 本场景埋下了哪些伏笔？
2. 本场景回收了哪些伏笔？
3. 伏笔的埋下和回收是否合理？
4. 是否有伏笔被遗忘？

**典型案例：**
```
伏笔（第 3 章）：
"摸金符突然发烫，像是在警告什么。"

遗忘（第 4-7 章）：
没有再提

问题：
- 伏笔埋下后没有回收
- 读者会忘记这个伏笔
- 浪费前期铺垫！

修复（第 8 章回收）：
"摸金符还在发烫。我想起第 3 章那次，也是这么烫。
那时候不知道是什么意思，现在明白了——它在警告我，
墓里的东西，不是我能掌控的。"
```

## 🚀 **工作流程**

### **完整 9 步流水线**

```
1. PLAN          - 场景规划（生成 SceneCard）
2. WRITE         - 正文写作
3. EXTRACT       - 事实抽取
4. VALIDATE      - 硬规则校验（H1-H7）
5. PATCH         - 硬规则修复
6. DEEP_REVIEW   - ⭐ 6 层深度审查
7. LOGIC_PATCH   - 深度问题修复
8. COMMIT        - 更新 World State
9. OUTPUT        - 输出最终版本
```

### **每层审查流程**

```
Layer 1: 信息流审查
  ↓
Layer 2: 视角一致性审查
  ↓
Layer 3: 情感弧线审查
  ↓
Layer 4: 动机深度审查
  ↓
Layer 5: 主题一致性审查
  ↓
Layer 6: 伏笔追踪审查
  ↓
汇总所有问题 → 自动修复 → 重新审查
```

## ⚙️ **配置选项**

### **调整审查深度**

```python
# 在 deep_logic_reviewer.py 中
temperature = 0.3  # 低温度 = 更严谨
# temperature = 0.5  # 中等 = 平衡
# temperature = 0.7  # 更高 = 更宽松
```

### **跳过某层审查**

```python
# 如果某层不需要，可以跳过
layers_to_skip = ["主题一致性审查", "伏笔追踪审查"]
```

### **自定义审查重点**

```python
# 在 prompt 中添加自定义审查点
custom_focus = [
  "检查是否有现代用语混入（时代一致性）",
  "检查盗墓专业术语是否准确"
]
```

## 📊 **Token 消耗分析**

### **单层审查**

- 提升作品质量显著

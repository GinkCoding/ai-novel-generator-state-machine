# 🧠 AI 驱动的逻辑审查 - 使用说明

## 🎯 **核心理念**

**不用写死的规则，让 AI 自己推理发现逻辑问题。**

---

## 🔄 **方法对比**

### **旧方法（写死规则）❌**

```python
rules = {
  "帝王墓": ["应该隐蔽", "多层防护"],
  "政府机构": ["保护人民", "不灭口"]
}

# 检查文本是否违反规则
if "墓门显眼" in text:
    issue("违反规则：帝王墓应该隐蔽")
```

**问题：**
- ❌ 只能发现已知问题
- ❌ 需要人工编写规则
- ❌ 无法适应新情况
- ❌ 规则越多越复杂

---

### **新方法（AI 推理）✅**

```python
prompt = """
请分析以下内容的逻辑问题：
- 因果推理：这个原因能推出这个结果吗？
- 常识推理：这符合常理吗？
- 动机推理：这个角色为什么会这样做？
- 一致性推理：前后矛盾吗？
"""

# AI 自动推理
result = ai_review(scene_card, draft, context)
```

**优势：**
- ✅ 能发现未知的逻辑问题
- ✅ 不依赖人工规则
- ✅ 适应各种情况
- ✅ 推理过程透明

---

## 📊 **工作原理**

### **输入**

```json
{
  "scene_card": {
    "ch": 8,
    "location": "墓门外门",
    "goal": "陈三爷讲述真相"
  },
  "draft": "正文内容（10000 字内）",
  "context": {
    "canon": {...},      // 世界观设定
    "world": {...},      // 当前状态

```

```

3. 检查常识合理性
   ↓
4. 推理角色动机
   ↓
5. 检查一致性
   ↓
6. 输出问题和推理过程
```

## 📋 **审查维度**

### **1. 因果推理**
- 这个原因能推出这个结果吗？
- 是否有未解释的事件？
- 因果链是否完整？

**示例问题：**
- "突然"发生的事件（缺乏原因）
- 原因不足以解释结果
- 因果跳跃

---

### **2. 常识推理**
- 这符合现实世界的常识吗？
- 是否符合时代背景？
- 是否符合物理规律？

**示例问题：**
- 1950 年代出现手机
- 帝王墓入口显眼
- 违反物理规律

---

### **3. 动机推理**
- 角色为什么会这样做？
- 动机是否充分？
- 行为是否符合动机？

**示例问题：**
- 反派为坏而坏（无动机）
- 角色行为与动机不符
- 组织行为违背其使命

---

### **4. 一致性推理**
- 前后是否矛盾？
- 设定是否自洽？
- 角色行为是否一致？

**示例问题：**
- 前面说 A，后面说非 A
- 双重标准
- 角色性格突变

---

### **5. 世界观推理**
- 是否符合故事设定的世界观？
- 是否符合社会背景？
- 是否符合组织性质？

**示例问题：**
- 政府机构行为不符合其性质
- 社会背景与时代不符
- 超自然元素无规则

## 📊 **输出解读**

### **severity 等级**

| 等级 | 说明 | 处理建议 |
|------|------|----------|
| **critical** | 严重逻辑漏洞 | 必须修复 |
| **major** | 重要逻辑问题 | 建议修复 |
| **minor** | 轻微逻辑瑕疵 | 可选修复 |

### **pass 判断**

- `pass: true` - 无 critical/major 问题
- `pass: false` - 有 critical/major 问题

## ⚠️ **注意事项**

### **1. AI 也会犯错**

- AI 的推理可能不正确
- 需要人工最终审核
- 不要盲目相信 AI

### **2. 允许艺术夸张**

- 小说不是科学论文
- 允许合理的艺术夸张
- 区分"不合逻辑"和"你不喜欢"

### **3. 上下文很重要**

- 提供完整的 canon/world/timeline
- 上下文越完整，审查越准确
- 缺少上下文会导致误报

---

## 🎯 **最佳实践**

1. **提供完整上下文** - canon + world + timeline
2. **审查后人工复核** - AI 建议，人类决定
3. **迭代改进** - 根据审查结果优化设定
4. **记录决策** - 为什么接受/拒绝某个问题

# 🧠 智能逻辑审查层 - 使用说明

## 📋 **问题反思**

之前的生成器为什么没发现明显的逻辑漏洞？

### **原流程的缺陷**

```
硬规则校验（validate.py）
├── ✅ 检查时间单调性
├── ✅ 检查禁止瞬移
├── ✅ 检查物品出现
└── ❌ 不检查常识合理性 ← 问题所在
```

**遗漏的检查：**
- ❌ 帝王墓应该隐蔽（常识）
- ❌ 701 不应该灭口自己人（动机）
- ❌ "自动开启"vs"需要钥匙"（矛盾）
- ❌ 国家队形象是否合理（世界观）

---

## 🆕 **新增功能**

### **8 步流水线（原 7 步）**

```
1. PLAN        - 场景规划
2. WRITE       - 正文写作
3. EXTRACT     - 事实抽取
4. VALIDATE    - 硬规则校验
5. PATCH       - 硬规则修复
6. LOGIC_REVIEW ⭐ 新增：智能逻辑审查
7. LOGIC_PATCH  ⭐ 新增：逻辑问题修复
8. COMMIT      - 更新 World State
9. OUTPUT      - 输出最终版本
```

### **2. 逻辑审查器**
`novel_ci/scripts/logic_reviewer.py`

**6 大检查：**
1. 常识合理性检查
2. 动机合理性检查
3. 世界观一致性检查
4. 因果链深度检查
5. 角色行为合理性检查
6. 设定自洽性检查

### **3. 更新生成器**
`core/state_machine_generator.py`

**新增方法：**
- `_logic_review()` - 执行逻辑审查
- `_patch_logic()` - 修复逻辑问题
- `_generate_output()` - 包含逻辑审查报告

## 🚀 **使用方法**

### **自动运行（推荐）**

生成章节时自动执行逻辑审查：

```bash
python3 test_chapter8.py
```

输出会显示：
```
🧠 Step 6/8: LOGIC_REVIEW - 智能逻辑审查
⚠️  发现 2 个逻辑问题
  - [CRITICAL] 常识合理性：帝王墓应该隐蔽
  - [MAJOR] 动机合理性：701 动机不合理

🔧 Step 7/8: LOGIC_PATCH - 逻辑问题修复
✓ 逻辑问题修复完成
```

### **手动运行**

```python
from novel_ci.scripts import logic_reviewer

result = logic_reviewer.review_logic(
    scene_card=scene_card,
    draft=draft_content,
    world_state=world,
    canon=canon
)

print(result['issues'])
```

## ⚠️ **注意事项**

### **1. 常识知识库需要完善**

当前版本包含基础常识，但需要持续添加：
- 更多历史常识（各朝代/各地区）
- 更多专业常识（考古/军事/政治）
- 更多人性常识（心理学/社会学）

### **2. 逻辑审查不是万能的**

仍然需要人工审查：
- 深层主题一致性
- 角色弧光完整性
- 情节节奏合理性

### **3. 误报处理**

如果逻辑审查误报：
- 在 `common_sense.json` 中添加例外规则
- 或调整审查器阈值
